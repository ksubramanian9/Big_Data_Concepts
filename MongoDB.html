Why MongoDB?

MongoDB is a document database built for developer productivity and horizontal scalability. Its flexible schema and rich query capabilities make it a popular choice for modern applications.

Guiding Principles

Schema Flexibility

Documents use a JSON-like structure, allowing fields to vary from record to record.

Horizontal Scale-Out

Sharding distributes data across clusters, enabling massive scale without complex joins.

Rich Query Capability

Queries support filtering, aggregation, geospatial data, and more with a consistent API.

Developer Friendly

Drivers for every major language and a natural data model speed up application development.

Core Concepts

Documents & Collections

Data is stored as documents inside collections rather than rows in tables.

Replica Sets

Multiple copies of data provide redundancy and automatic failover.

Indexes

Secondary indexes accelerate queries on frequently accessed fields.

Query Cursors

Each find() operation returns a cursor that lazily iterates matching documents, allowing applications to page through large result sets efficiently.

Aggregation Pipeline

Stages transform documents for analytics, similar to SQL GROUP BY operations.

Sharding

Data is partitioned across shards, each holding a subset of the dataset.

Transactions

Multi-document ACID transactions ensure data consistency when needed.

Aggregation Framework

MongoDB's aggregation framework processes documents through a pipeline of stages, letting you reshape, filter, group, and analyze data in-place without exporting it to an external analytics tool.

How the Pipeline Works

Each stage receives documents from the previous stage, transforms them, and passes them along. Common stages include $match to filter documents, $group to aggregate values, $sort to order results, and $project to reshape the output. MongoDB executes the pipeline on the server and can leverage indexes on early stages.

Sample Pipeline
db.orders.aggregate([
  { $match: { status: "delivered", orderDate: { $gte: ISODate("2024-01-01") } } },
  { $group: {
      _id: "$customerId",
      totalSpent: { $sum: "$orderTotal" },
      averageBasket: { $avg: "$orderTotal" },
      orders: { $sum: 1 }
  } },
  { $sort: { totalSpent: -1 } },
  { $project: {
      customerId: "$_id",
      totalSpent: 1,
      averageBasket: 1,
      orders: 1,
      _id: 0
  } }
]);

This pipeline finds 2024 orders that were delivered, groups them per customer, ranks them by spending, and returns a tidy report for dashboards.

Popular Stages & Uses
  • $unwind – Flattens array values for per-item analysis.
  • $addFields – Creates computed columns, e.g., profit margins.
  • $lookup – Performs left outer joins against other collections.
  • $facet – Runs multiple pipelines in parallel to serve dashboards.

Indexes in MongoDB

Indexes keep frequently queried fields in sorted data structures so MongoDB can locate matching documents quickly. Without indexes, the server must scan every document in a collection.

Why Indexes Matter

  • Accelerate read queries and allow query planners to avoid full collection scans.
  • Support efficient sorting and range queries by maintaining ordered keys.
  • Enable unique constraints that enforce data quality (e.g., unique emails).
  • Improve performance of aggregation pipelines when early stages filter on indexed fields.
Creating and Inspecting Indexes
db.users.createIndex({ email: 1 }, { unique: true });
db.orders.createIndex({ customerId: 1, orderDate: -1 });
db.orders.getIndexes();

This example adds a unique single-field index and a compound index used for "latest orders per customer" queries.

Index Types at a Glance

Single Field

Ordered list of one key (ascending or descending). Ideal for equality or range filters on a single field.

name ↑

Compound

Combines multiple fields and preserves their sort order. Useful for queries that filter by a prefix of the indexed fields.

customerId ↑ orderDate ↓

Multikey

Automatically created for array fields so each element is indexed. Perfect for tag lists or categories.

tags[0] tags[1] ...

Text

Supports language-aware text search with stemming and scoring.

"analytics" "cloud" score

Geospatial & Hashed

2dsphere indexes enable map queries within polygons or by distance; hashed indexes evenly distribute shard keys for write scaling.
